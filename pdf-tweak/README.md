# PDF Tweak

对现有 PDF 进行精准、小幅度的文本编辑——改一个错字、替换一个词、追加一段标注。通过直接修改 PDF 内容流（content stream）实现，不依赖重渲染。

## 主要功能

| 操作 | 说明 | 示例 |
|------|------|------|
| **查找文本** | 定位文本在内容流中的精确字节位置，返回字体、CID 序列、坐标 | 找出 PDF 中所有 `"hello"` 的位置 |
| **替换文本** | 同字体替换，修改后文件结构不变 | `"hello"` → `"world"` |
| **删除文本** | 删除文字并清理相邻的 kerning 值 | 删除多余的空格或标点 |
| **追加文本** | 在目标文字后面追加内容，自动处理字体切换和 TJ 数组拆分 | `"Company"` 后追加 `" (edited)"` |
| **扫描字体覆盖** | 一次性检查页面上所有字体子集对目标字符的覆盖情况 | 找到能渲染 `"✓"` 的字体 |

## 应用场景

- **修复简历错字**：PDF 源文件丢失，需要改一个字或更新日期
- **合同/协议微调**：修改条款编号、更新公司名称
- **表单填充后的修正**：填写完 PDF 表单后发现有个字段写错了
- **批量水印/标注**：在文档中特定文字后追加注释标记
- **隐私脱敏**：将文档中的真实姓名替换为占位符再分享

## 快速开始

```bash
# 1. 定位目标文字
python scripts/edit_pdf.py report.pdf 0 --find "Acme Corp"

# 2. 检查替换文字是否在字体中
python scripts/font_map.py report.pdf 0 --scan-all --chars "Acme Inc"

# 3. 执行替换（先 dry-run 预览）
python scripts/edit_pdf.py report.pdf 0 \
  --replace "Acme Corp" --with "Acme Inc" --dry-run
python scripts/edit_pdf.py report.pdf 0 \
  --replace "Acme Corp" --with "Acme Inc"

# 4. 追加标注（跨字体）
python scripts/edit_pdf.py report.pdf 0 \
  --insert-after "Acme Inc" --insert-text " (updated)" --font FONT2
```

## 工作原理

PDF 中的文字不是"字符"，而是"在特定位置用特定字体画特定字形"的绘图指令。本工具直接修改这些指令：

```
原始内容流:                                  修改后:
BT                                          BT
  /F1 12 Tf      ← 选字体                     /F1 12 Tf
  [<0048>0<0065>0<006C>0<006C>0<006F>] TJ   [<0048>0<0069>] TJ    ← "Hi"
  ← "Hello"                            ET
ET
```

实际操作步骤：

1. **映射**：用 pdfplumber 提取文字，获取目标文本的页码、坐标和字体名
2. **验证字体**：检查替换/追加的每个字符是否在字体子集中（字体子集只包含 PDF 中使用过的字符）
3. **定位**：在内容流二进制中搜索 CID 序列，找到精确的字节偏移
4. **修改**：直接替换/插入/删除字节，写回 PDF

## 技术架构

```
pdf-tweak/
├── SKILL.md                  # 工作流定义（给 AI agent 看）
├── README.md                 # 本文档（给人看）
└── scripts/
    ├── font_map.py           # 字体 CMap 分析工具
    │   • 解析 ToUnicode CMap（Unicode ↔ CID 映射表）
    │   • --scan-all：一次性扫描页面上所有字体的字符覆盖
    │   • 输出覆盖率排名，推荐最佳字体
    │
    └── edit_pdf.py           # 内容流编辑工具
        • --find：定位文本的字节偏移和 CID 序列
        • --replace：同字体文本替换
        • --delete：删除文本
        • --insert-after：跨字体追加，自动拆分 TJ 数组
        • --dry-run：预览即将修改的字节（不写入文件）
        • --match-index / --bounds：多匹配时精确定位
```

### 核心概念

**CID 字体与内容流**

```
PDF 文件结构:
┌─────────────────────────────────┐
│ Page Object                     │
│  /Resources                     │
│    /Font                        │
│      F1: CIDFontType2           │  ← 字体对象（含字形数据）
│        /ToUnicode CMap          │  ← Unicode ↔ CID 映射
│        /FontFile2 (TTF subset)  │  ← 嵌入的字体二进制
│  /Contents                      │
│    BT...ET  (text blocks)       │  ← 内容流（绘图指令）
│      [<CID> kerning <CID>] TJ  │  ← TJ 数组（字形+间距）
└─────────────────────────────────┘
```

**TJ 数组格式**

```
[<CID1> kerning <CID2> kerning ... <CIDn>] TJ
  │       │
  │       └─ 数字：字间距调整（千分之一 em）
  └─ 十六进制 CID：字体中的字形编号
```

**`--insert-after` 如何实现跨字体追加**

```
原始:  [<CIDa>0<CIDb>0<CIDc>0<CIDd>] TJ
  
追加后:
       [<CIDa>0<CIDb>] TJ              ← 前半部分（原字体）
       /NewFont 12 Tf [<CIDx>0<CIDy>] TJ  ← 新文字（新字体）
       /OldFont 12 Tf [0<CIDc>0<CIDd>] TJ ← 后半部分（切回原字体）
```

## 优势

- **不重渲染**：直接修改原始内容流，不经过任何渲染引擎，不会改变排版、字体嵌入、图片质量
- **精准**：操作粒度精确到单个字符，只改你指定的文字，其余内容原封不动
- **跨字体支持**：`--insert-after` 可以在一行文字中使用不同字体追加内容，自动处理字体切换
- **无依赖膨胀**：只依赖 `pypdf` 和 `pdfplumber` 两个轻量 Python 库
- **可预览**：所有修改操作都支持 `--dry-run`，字节级对比修改前后的内容流
- **AI 友好**：以 skill 形式组织，AI agent 可以直接按照工作流执行 PDF 编辑任务

## 局限与缺陷

### 字体子集化限制（核心限制）

PDF 字体是**子集化的**——只有文档中实际使用过的字符才被嵌入。如果想把 `"Java"` 改成 `"Java8"`，但 `8` 从未在文档中出现过，则字集中不存在该字形，操作会失败。

```
字体子集示意图:
完整 Calibri 字体 (2000+ 字符)  →  PDF 嵌入子集 (仅使用过的字符)
├─ A ✓                             ├─ A ✓
├─ B ✓                             ├─ B ✓
├─ C ✓              PDF 生成时     ├─ C ✓
├─ 8 ✓              ──────────→    ├─ 8 ✗  (没用到，被裁掉了)
├─ ...                             ├─ ...
```

**缓解措施**：`--scan-all` 可以帮你找到页面上哪些字体包含了所需字符，然后通过 `--insert-after --font` 跨字体完成操作。

### 只支持 CID 字体

工具只处理使用 **CID 字体 + TJ 操作符** 的 PDF。这类 PDF 通常由以下工具生成：
- 中文文字处理软件（WPS、Microsoft Word 中文版）
- LaTeX + CJK 宏包
- 浏览器打印生成的 PDF（部分）
- 在线简历/文档生成器

以下 PDF 不受支持：
- 使用 Type1/TrueType 简单字体 + `Tj` 操作符的 PDF（如英文 Word 文档、reportlab 生成的 PDF）
- 扫描版 PDF（文字是图片，不是文本指令）

### 不支持的操作

| 不支持 | 原因 |
|--------|------|
| 修改字体大小/颜色/样式 | 需要解析字体资源和图形状态，远超内容流字节替换的范围 |
| 添加页面/合并 PDF | 属于 PDF 生成/拼接，不在本工具范围内 |
| 编辑图片/表格 | 图片和表格不是文本内容流 |
| 向字体子集添加新字形 | 需要字体工程：解析 OpenType 表、重新编码字体流 |
| 跨页文字重排 | PDF 没有"段落"概念，每个字符的位置是绝对坐标 |

### 其他注意事项

- **中文括号 vs 英文括号**：如果半角 `()` 不在字集中，可以尝试全角 `（）`，通常 CJK 字体都包含
- **坐标精度**：用 `--bounds` 筛选多匹配时，pdfplumber 和内容流的坐标体系可能有微小偏差
- **修改后文件大小**：可能略有变化（几个字节），内容流长度改变会影响交叉引用表

## 与其他 PDF 工具的对比

| | PDF Tweak | Adobe Acrobat | 重渲染方案 (reportlab 等) |
|---|---|---|---|
| 修改方式 | 直接 patching 内容流 | 完整编辑器 | 重新生成整个 PDF |
| 对原文件的影响 | 最小（仅改目标字节） | 大面积重写 | 完全重建 |
| 字体保真度 | 100%（不改变字体） | 高 | 取决于替换字体 |
| 操作范围 | 替换/删除/追加文字 | 全面编辑 | 全面 |
| 技术门槛 | 命令行 + Python | GUI | 编程 |
| 适合场景 | 小修小改、自动化 | 人工编辑 | 从零生成 |
